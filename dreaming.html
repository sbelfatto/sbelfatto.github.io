When introducing a new technology or design, why did Frederick Brooks advise “plan to throw one away?”

Brooks believes you will not get it right the first time so it is best to plan to throw one away. I could not agree more. I never get a program right the first time. Trial and error. I also feel like you learn more when you break something.

What is a “core” dump? Why the use of the word core?

The word core is used because the commercial computers of the 50's and 60's memory banks were built out of wound wire coils known as ferrite cores. A core dump is a report of the contents of acomputers memory at the moment of failure. Giving a lot of clues for bug hunters. Core dumps usually happen when there is an "irreconciliable conflict or unexpected error."

What is “yak-shaving?”

Eric Raymond defines it as, "seemingly pointlessactivity which is actually necessary to solve a problem which solves a problems which several levels of recusion later, solves the real problem you're working on." I think of it as tooling. you have a task that has a set of sub tasks and you create a tool that makes those subtasks faster or easier, thus making the task faster.

What is term “edge cases” referring to in software development?

Edge cases are obscure unlikely choices that any software allows. This is when a programmer thinks of every possible scenario you can get into using this software.

Summarize briefly Linus Torvalds advice about “large projects” give in 2004

He said to not start them. Torvalds believes in working on small trivial projects focusing on the details and not expecting anything to get done fast. He says if you start a project that you plan on being large you most likely will fail. he says to never ever dream of the big picture.
